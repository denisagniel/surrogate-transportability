# 2026-07-13 — Generality-validation study

## Summary
Diagnosed the canonical-validation cluster timeouts (median walltime sizing;
dgp1 ~2x slower than boundary DGPs). Then, at the user's push for a study that
convincingly shows the estimator works IN GENERAL (not 4 hand-picked DGPs),
designed + built a new `simulations/generality-validation/` study and ran a full
Phase 0 pilot to de-risk it.

## Key findings (Phase 0)
- **Exact truth for any DGP**: `Theta = cor(Delta_S, Delta_Y)`, `Delta(Q)=sum_k q_k tau(k)`
  depends only on cell CATEs, not the error law — so random DGPs (any error law)
  have an exact `rho_true` with no noisy reference run. Validated.
- **The estimator's bias is finite-n attenuation that collapses with n** (both IW
  and AIPW). dgp1: bias -0.29 @ n=2000 -> -0.014 @ n=10000, coverage 0.90->0.95.
  It's a large-n method (matches n=o(M) theory). Not a defect.
- **Fit-once AIPW nuisances (Mode 1) valid + feasible** (~3.6x faster, ~unbiased
  at n=10000). Mode-2 (refit per Q) is NOT a bug — it's the deferred
  non-X-transportability sensitivity analysis.
- **Jackknife bias-correction works ensemble-wide**: HALVES bias at n=2000
  (26/30 random DGPs improved), HARMLESS at n=10000. EIV moment-subtraction and
  SIMEX both rejected (harmful/overshoot).

## Built (Phase 1, wave 1 = IW/RCT)
`simulations/generality-validation/` — random ensemble (rho-balanced) + canonical
anchors + n-scaling slice; every unit reports raw AND jackknife-corrected rho.
Profiler patched to worst-case per-config sizing (fixes the timeout; per-unit
time varies ~70x across the n-grid). Smoke-tested end-to-end.

## Decisions
- Wave 1 IW only; AIPW deferred to wave 2 (fit-once prototyped).
- Compute both raw + jackknife every rep.
- Ensemble rho was boundary-piled (65% |rho|>0.9); fixed via DGP tweak +
  balanced-seed rejection sampling.
