## Key Design Decisions

### Fresh Innovations for Gradient
**Decision**: Draw fresh M innovations at each of 4 gradient evaluation points (total 4M).

**Why**: Reusing innovations makes H constant in (δ_S, δ_Y) because cor(X,Y) is shift-invariant.

**Cost**: 4x computational cost for gradient, but essential for correctness.

### Epsilon = 0.01
**Decision**: Use ε = 0.01 (1% perturbation) instead of 1e-6.

**Why**:
- H involves Monte Carlo estimation
- Var(∇̂H) ≈ σ²/(2ε²M)
- Small ε → unstable (ε=1e-6 → gradient magnitude ~100,000)
- ε=0.01 → stable (gradient magnitude ~1-10)

**Evidence**: Tested across ε ∈ {0.01, 0.005, 0.001, 1e-4, 1e-5, 1e-6}.

## Technical Notes

### Mathematical Structure
φ(F_λ) = H(Δ_S(P₀), Δ_Y(P₀))

where H(δ_S, δ_Y) = E_μ[φ((1-λ)δ_S + λΔ_S(P̃), (1-λ)δ_Y + λΔ_Y(P̃))]

### Asymptotic Theory
√n(φ̂_n(λ) - φ(F_λ)) → N(0, σ²(λ))

where σ²(λ) = (∇H)ᵀ V(λ) (∇H)

- V(λ): 2×2 covariance matrix from influence functions
- ∇H: Numerical gradient via central differences

### Gradient Computation
- ∂H/∂δ_S = [H(δ_S+ε, δ_Y) - H(δ_S-ε, δ_Y)] / (2ε)
- ∂H/∂δ_Y = [H(δ_S, δ_Y+ε) - H(δ_S, δ_Y-ε)] / (2ε)
- Each H evaluation: M fresh innovations
- Total: 4M innovations for gradient + M for point estimate = 5M

## Critical Conceptual Fix: Reweighting vs Resampling

### Problem Discovered

After implementing simulation validation, initial tests showed **0% coverage** in under-model validation. This revealed a fundamental misunderstanding about what the method measures.

**Initial (incorrect) validation approach:**
- Generated future studies by **resampling** to create independent datasets
- Each future study: new random sample from mixture distribution
- Result: ~0 correlation (independent samples have no correlation)
- Coverage: 0% because method showed ~0.66 correlation but truth was ~0

**This seemed to show the method was broken, but actually the validation was wrong!**

### The Key Insight

**What should we be measuring?** After reading the paper carefully (lines 68-76, 225-237):

The method measures **correlation of treatment effects across population mixtures, net of sampling variability**.

Under finite support (lines 109-114), Q = (1-λ)P₀ + λP̃ is a **mixture of probability distributions**, not a resampling instruction.

For a probability distribution Q:
```
Δ_Y(Q) = E_Q[Y(1) - Y(0)]
       = Σᵢ qᵢ Yᵢ (weighted average, not resample!)
```

### Reweighting vs Resampling

**Reweighting** (correct):
- Draw Dirichlet weight vectors
- Compute treatment effects by applying weights to **same data**
- Measures: correlation across population compositions
- Net of sampling variability (focuses on population heterogeneity)

**Resampling** (incorrect for this purpose):
- Draw Dirichlet weights
- Generate new samples from mixture
- Includes sampling variability
- Kills correlation (independent samples)

### User Guidance

> "The intended use case is for genuinely new populations, though we want to control how different they are. Correlation shouldn't necessarily be near 0 just because studies are independent - if S is a good surrogate, treatment effects should be related across similar populations."

> "It should be measuring correlation of treatment effects net of sampling variability."

This clarified that we want **population-level correlation**, not correlation including sampling noise.

### The Fix

**Corrected validation** (`quick_test_under_model_corrected.R`):
```r
# Both truth and method use REWEIGHTING
for (m in 1:M) {
  # Draw innovation weights
  p_tilde <- rdirichlet(1, rep(1, n))
  
  # Mixture weights
  q_weights <- (1 - lambda) * (1/n) + lambda * p_tilde
  
  # Treatment effects via reweighting (NOT resampling)
  delta_s <- compute_treatment_effect_weighted(baseline, "S", q_weights)
  delta_y <- compute_treatment_effect_weighted(baseline, "Y", q_weights)
}
true_correlation <- cor(delta_s_vec, delta_y_vec)
```

**Method was already correct!** It was using reweighting all along via `compute_treatment_effect_weighted()`.
