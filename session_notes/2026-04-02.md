## Fix Multivariate Coverage Issue in Wasserstein Minimax IF-Based Inference

**Status:** ✅ IMPLEMENTED AND VALIDATED

---

### Problem Identified

Wasserstein minimax IF-based inference showed 87.8% coverage for 2 covariates (vs 95% target), but 94-96% coverage for 1 covariate.

**Root cause:** Cost matrices used squared Euclidean distance that scales linearly with dimension WITHOUT normalization:
- 1D: Mean cost ≈ 2
- 2D: Mean cost ≈ 4
- This caused oracle truth to be systematically inflated in multivariate case
- Estimator couldn't match inflated oracle → poor coverage

---

### Solution Implemented

**Fix:** Normalize cost matrices by number of covariates:

```r
d <- ncol(X)
costs <- rowSums((X - matrix(X[j, ], ...))^2) / d
```

This makes costs represent **average squared difference per dimension**.

### Key Insights

1. **Dimension scaling matters:** Cost matrices must be normalized when comparing across different covariate dimensionalities

2. **Self-consistency is critical:** Oracle truth and estimator must use the same cost normalization

3. **Theoretical justification:** For standard normal covariates, `E[||X_i - X_j||²/d] ≈ 2` is constant across dimensions, making the penalty scale dimension-invariant

4. **No breaking changes:** Normalization by `/1` in univariate case has no adverse effects
