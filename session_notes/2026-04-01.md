# Session Notes: 2026-04-01

## Goal

Implement Wasserstein minimax concordance estimation with valid influence function-based confidence intervals.

## Problem

- **Initial state:** IF-based CIs for nested expectations with estimated nuisances had poor coverage (56%)
- **Working case:** Simple concordance E[τ_S × τ_Y] with cross-fitting had correct coverage (94%)
- **User insight:** "Cross-fitting isn't the issue - it works with E(tau_s tau_y). I think there's just an error in the EIF."

## Solution

### Error Identified

The nuisance term in the IF had an **incorrect (1/n) factor**:

```r
# WRONG:
term3 <- (1/n) * sum(W[k, ]) * IF_h_k

# CORRECT:
term3 <- sum(W[k, ]) * IF_h_k
```

### Why This is Correct

**Concordance benchmark:** For Ψ = E[h(X)], the nuisance terms have no explicit 1/n:
```
IF = h(X) - Ψ̂ + τ_Y(X) × IF_τ_S + τ_S(X) × IF_τ_Y
```

**Nested case:** For Ψ = E_X[φ(X)] where h(X_k) affects all φ(X_j):
- Total sensitivity: Σ_j W[k,j] (aggregated over all reference points)
- No additional 1/n scaling needed

**Validation:** Empirically tested four scalings; only removing (1/n) gave correct coverage.

## Key Insights

**[LEARN:IF-derivation]** When deriving IF nuisance terms:
1. Use concordance E[h(X)] as benchmark (no 1/n in nuisance terms)
2. For nested functionals, use aggregated sensitivity directly
3. Intermediate (1/n) factors from empirical means cancel in final IF
4. Always validate empirically with different scalings

**[LEARN:cross-fitting]** Cross-fitting works correctly when IF is correct. Poor coverage indicates IF error, not cross-fitting issue.

## Complete IF Formula

For Ψ = E_X[-τ log E_{X'}[exp(-(h(X') + γC(X,X'))/τ)]] with estimated h = τ_S(X) × τ_Y(X):

```
IF(O_k) = term1 + term2 + term3

term1 = -τ log m(X_k) - Ψ̂                          # Outer
term2 = -τ Σ_j [g(X_j,X_k)/m(X_j)]/n + τ           # Inner
term3 = Σ_j W[k,j] × IF_h_k                         # Nuisance (CORRECTED)

where IF_h_k = τ_S(X_k) × IF_τ_Y + τ_Y(X_k) × IF_τ_S
```
