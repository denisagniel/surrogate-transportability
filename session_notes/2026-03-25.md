## Phase 2: Wasserstein Minimax Algorithm ✅

**Files Created:**
- `package/R/wasserstein_minimax.R` (450 lines)
  - `sample_wasserstein_perturbation()` - Generate q in W-ball
  - `estimate_minimax_single_scheme_wasserstein()` - Single scheme
  - `estimate_minimax_ensemble_wasserstein()` - Multi-scheme ensemble

**Algorithm:**
```
For each discretization scheme:
  1. Discretize into J types
  2. Compute type centroids in covariate space
  3. Construct cost matrix C[i,j] = ||centroid_i - centroid_j||²
  4. Sample M distributions q_m satisfying W₂(q_m, p₀) <= λ_W
  5. Compute treatment effects via deterministic reweighting
  6. Compute functional from effects distribution
Take MINIMUM across schemes (ensemble estimate)
```

## Validation & Correction (Critical Update)

### Pre-Commit Validation Revealed Critical Bug

Comprehensive validation testing revealed that W_2 distances were **always returning 0**:

**Root Cause:**
- Used approximation: W_2^2 ≈ (q-p₀)'C(q-p₀)
- Requires cost matrix C to be positive semi-definite
- **C is NOT PSD** for arbitrary type centroids (has negative eigenvalues)
- Result: All W_2 distances computed as 0, no ball exploration

**Evidence:**
```
Cost matrix eigenvalues: [24.21, 0.00, 0.00, -1.36, -9.44]
                                                  ^^^   ^^^
(q-p₀)'C(q-p₀) = -0.081 (negative!)
W_2 = sqrt(max(0, -0.081)) = 0 (always zero)
```

### The Fix (Option 1: Proper OT)

Replaced approximation with proper optimal transport solver:

**Implementation:**
- Uses `transport::wasserstein()` for exact LP solution
- Fallback to `lpSolve` if transport unavailable
- Note: `transport::wasserstein(p=2)` returns W_2^2, so take sqrt()

**Trade-off:**
- ~40x slower (~0.04s vs ~0.001s per sample)
- **BUT: Mathematically correct vs fundamentally broken**
- User requested correctness over speed ✓

## Sampling vs Closed-Form Trade-off Analysis ✅

**User insight:** "Sampling gives distributional information, not just minimum"

**Critical trade-off identified:**

**Closed-form (concordance):**
- ✓ 10-500× faster
- ✓ Gives minimum (worst-case) and maximum analytically
- ✗ No mean, median, quantiles, variance
- ✗ No distributional shape information
- ✗ No risk profiling capabilities

**Sampling (correlation or concordance):**
- ✗ 10-500× slower
- ✓ Full distribution over Q ∈ B_λ(P₀)
- ✓ All summaries: min, 5th/25th/75th/95th percentiles, median, mean, max, variance
- ✓ Risk profiling for different decision contexts
- ✓ Robustness assessment: narrow distribution = stable, wide = fragile

**Recommended hybrid approach:**
1. Screen with closed-form (fast) for many λ values
2. Identify critical λ (near decision boundaries)
3. Detailed distributional analysis at critical λ with sampling (rich info)

## Lessons Learned

**[LEARN:paper-positioning]** Lead with the gap you fill, not the technique you use. "Evaluating transportability" is the contribution; "minimax" is the technique. The current paper leads with technique when it should lead with contribution.

**[LEARN:methods-comparison]** When comparing to existing methods, emphasize complementarity (different questions) not superiority (better assumptions). This positions the work as filling a gap rather than replacing established approaches.

**[LEARN:computational-innovation]** Computational improvements that enable new applications are scientific contributions, not just optimizations. 9-487× speedup enables real-time monitoring and large-scale sensitivity—these are new research directions, not just "faster code."

**[LEARN:trade-off-documentation]** When an innovation involves trade-offs (speed vs information), document both sides honestly. Users need guidance on when to use what. "Always use X" is rarely the right answer.

**[LEARN:empirical-evidence]** Conceptual comparisons (Table 1 showing assumptions) are useful but insufficient. Show empirical performance differences (coverage under violations) to make the contribution concrete and undeniable.
