## Phase 2: Wasserstein Minimax Algorithm ✅

**Files Created:**
- `package/R/wasserstein_minimax.R` (450 lines)
  - `sample_wasserstein_perturbation()` - Generate q in W-ball
  - `estimate_minimax_single_scheme_wasserstein()` - Single scheme
  - `estimate_minimax_ensemble_wasserstein()` - Multi-scheme ensemble

**Algorithm:**
```
For each discretization scheme:
  1. Discretize into J types
  2. Compute type centroids in covariate space
  3. Construct cost matrix C[i,j] = ||centroid_i - centroid_j||²
  4. Sample M distributions q_m satisfying W₂(q_m, p₀) <= λ_W
  5. Compute treatment effects via deterministic reweighting
  6. Compute functional from effects distribution
Take MINIMUM across schemes (ensemble estimate)
```

## Key Design Decisions

1. **Type-level approach:** J << n for efficiency (consistent with TV approach)
2. **Constrained sampling:** Primary projection method (no heavy dependencies)
3. **Euclidean default cost:** Simple, interpretable
4. **Separate function:** Preserves TV approach, allows comparison
5. **API consistency:** Mirrors `surrogate_inference_minimax()` structure

## Wasserstein vs TV-Ball

| Feature | TV-Ball | Wasserstein Ball |
|---------|---------|------------------|
| Constraint | TV(Q, P₀) ≤ λ | W₂(Q, P₀) ≤ λ_W |
| Use Case | Arbitrary shifts | Covariate shift |
| Tightness | More conservative | Tighter under cov shift |
| Interpretation | General robustness | Covariate space distance |

## Validation & Correction (Critical Update)

### Pre-Commit Validation Revealed Critical Bug

Comprehensive validation testing revealed that W_2 distances were **always returning 0**:

**Root Cause:**
- Used approximation: W_2^2 ≈ (q-p₀)'C(q-p₀)
- Requires cost matrix C to be positive semi-definite
- **C is NOT PSD** for arbitrary type centroids (has negative eigenvalues)
- Result: All W_2 distances computed as 0, no ball exploration

**Evidence:**
```
Cost matrix eigenvalues: [24.21, 0.00, 0.00, -1.36, -9.44]
                                                  ^^^   ^^^
(q-p₀)'C(q-p₀) = -0.081 (negative!)
W_2 = sqrt(max(0, -0.081)) = 0 (always zero)
```

### The Fix (Option 1: Proper OT)

Replaced approximation with proper optimal transport solver:

**Implementation:**
- Uses `transport::wasserstein()` for exact LP solution
- Fallback to `lpSolve` if transport unavailable
- Note: `transport::wasserstein(p=2)` returns W_2^2, so take sqrt()

**Trade-off:**
- ~40x slower (~0.04s vs ~0.001s per sample)
- **BUT: Mathematically correct vs fundamentally broken**
- User requested correctness over speed ✓
