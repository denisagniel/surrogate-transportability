# Session Notes: 2026-05-11

## Goal
Fix cluster result combination script to handle DGP 5's undefined PTE values (antisymmetric effects where ΔY ≈ 0).

## Problem
`cluster/slurm/combine_results.R` was failing with error:
```
Error in mean(df$PTE_hat, na.rm = TRUE) - PTE_true :
  non-numeric argument to binary operator
```

DGP 5 has `PTE_P0: NaN` in YAML config (ΔY ≈ 0 makes PTE undefined), but combine script tried to:
1. Compute bias as `mean(PTE_hat) - PTE_true` where `PTE_true = "NaN"` (string from YAML)
2. Print percentage bias with `100 * bias / PTE_true` (division by "NaN" string)

## Solution Applied

**File:** `cluster/slurm/combine_results.R`

**Changes:**
1. Added dgp5 to processing list (line 18)
2. Added NaN/NA handling for `PTE_true` (lines 85-90):
   - Detect YAML "NaN" string
   - Convert to `NA_real_`
   - Same pattern as `run_single_replication.R` fix (commit 3be0e7a)
3. Conditional bias computation (lines 101-105):
   - Only compute `bias_PTE` if `PTE_true` is not NA
   - Otherwise set `bias_PTE = NA_real_`
4. Conditional reporting (lines 120-130):
   - Print "NA (undefined)" when `PTE_true` is NA
   - Skip percentage calculation
5. Conditional summary report (lines 176-181):
   - Handle NA PTE in final summary table

**Code pattern:**
```r
# Handle NaN/NA in PTE
pte_val <- dgp_config$PTE_P0
if (is.character(pte_val) && pte_val == "NaN") {
  PTE_true <- NA_real_
} else {
  PTE_true <- as.numeric(pte_val)
}

# Conditional bias
if (!is.na(PTE_true)) {
  bias_PTE <- mean(df$PTE_hat, na.rm = TRUE) - PTE_true
} else {
  bias_PTE <- NA_real_
}

# Conditional printing
if (is.na(PTE_true)) {
  cat(sprintf("    TRUE PTE = NA (undefined)\n"))
  cat(sprintf("    Mean PTE_hat = %.4f (SD = %.4f)\n", ...))
  cat(sprintf("    Bias = NA (PTE undefined)\n"))
} else {
  cat(sprintf("    TRUE PTE = %.4f\n", PTE_true))
  cat(sprintf("    Mean PTE_hat = %.4f (SD = %.4f)\n", ...))
  cat(sprintf("    Bias = %.4f (%.1f%%)\n", bias_PTE, 100 * bias_PTE / PTE_true))
}
```

## Fifth Session: Implement AIPW Support in Adaptive Function

### Goal
Extend `tv_ball_correlation_IF_adaptive.R` with AIPW (doubly robust) support for observational studies with confounding.

### Context
- Current function supports `"bootstrap"` and `"importance_weighting"` methods
- Both assume RCT setting (e(X) = 0.5)
- Need AIPW for observational studies with estimated nuisances (propensity scores + outcome regressions)
- Will support AIPW robustness simulation study (263 settings × 1000 reps)

### Implementation

**Phase 1: Extended Function Signature**

Added parameters to support two AIPW modes:

**External mode** (user provides pre-estimated nuisances):
- `e_hat` - Propensity scores P(A=1|X)
- `mu_1_S`, `mu_0_S` - Outcome regressions E[S|A=a,X]
- `mu_1_Y`, `mu_0_Y` - Outcome regressions E[Y|A=a,X]

**Cross-fitting mode** (function estimates internally):
- `method_e = c("linear", "gam", "rf")` - Propensity score method
- `method_mu = c("linear", "gam", "rf")` - Outcome regression method
- `n_folds = 5` - Number of cross-fitting folds

**Phase 2: Pre-Loop Setup** (lines 163-185)

Mode detection and initialization:
```r
if (method == "aipw") {
  if (!is.null(e_hat)) {
    # MODE 1: External nuisances
    use_external_nuisances <- TRUE
  } else {
    # MODE 2: Cross-fitting
    folds <- sample(rep(1:n_folds, length.out = n))
    # Pre-allocate storage for M_max nuisance estimates
    e_hat_all <- matrix(0, nrow = n, ncol = M_max)
    mu_1_S_all <- matrix(0, nrow = n, ncol = M_max)
    # ... etc
    use_external_nuisances <- FALSE
  }
}
```

**Phase 3: Treatment Effect Computation** (lines 250-354)

AIPW estimator with importance weighting:
```r
# Compute importance weights w_i = Q_m(X_i) / P_0(X_i)
w_i <- Q_m[k_i] / P0_categorical[k_i]

# AIPW: IPW + regression correction
aipw_S <- w_i * (
  A * (S - mu_1_S_m) / e_hat_m -
  (1-A) * (S - mu_0_S_m) / (1 - e_hat_m) +
  mu_1_S_m - mu_0_S_m
)

Delta_S_batch[m] <- mean(aipw_S)
```

Cross-fitting mode fits nuisances via k-fold CV with Q_m-weighted data, supports linear/GAM/RF methods.

**Phase 4: Influence Function Computation** (lines 428-471)

Three-term AIPW influence function:
```r
psi_S[i, m] <- w_i[i] * (
  A[i] * (S[i] - mu_1_S_m[i]) / e_hat_m[i] -
  (1-A[i]) * (S[i] - mu_0_S_m[i]) / (1 - e_hat_m[i]) +
  mu_1_S_m[i] - mu_0_S_m[i] - Delta_S_m
)
```
