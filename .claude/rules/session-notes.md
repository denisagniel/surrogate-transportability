# Session Notes

**Location:** `<project_root>/session_notes/` (in the repo the user is working in)
**Naming:** `YYYY-MM-DD.md` (append to today's file if it exists)
**Template:** `templates/session-note.md`

## Same three triggers as session logging

When you update the session log (see `session-logging.md`), **also** update the project's session note:

1. **Post-plan** — Create or append to `session_notes/YYYY-MM-DD.md`: goal, approach, key context.
2. **Incremental** — Append 1–3 lines to today's session note when: a decision is made, a problem is solved, the user corrects something, or the approach changes.
3. **End-of-session** — Add summary, key changes, remaining work, decisions (daily-note-ready).

## Content

Keep session notes concise and daily-note-ready: summary, key changes, decisions, open questions, next steps. Session notes feed **daily notes**, which live outside this repo at `$AGENT_ASSISTED_RESEARCH_META_NOTES` (or the path in meta-spec/META_PROJECT_NOTES.md). See that doc for the meta-project notes structure.
